## 1. REQ-MOBILE-007: 온디바이스 Whisper 모델 관리

### AC-007-01: 모델 다운로드 및 저장

**Given** 앱이 처음 설치되어 로컬에 모델 파일이 없다
**When** 사용자가 오프라인 STT 기능에 접근한다
**Then** 시스템은 모델 다운로드 안내 다이얼로그를 표시한다
**And** 사용자가 다운로드를 승인하면 whisper-base 모델(~150MB)을 다운로드한다
**And** 다운로드 완료 후 SHA-256 체크섬 검증을 통과하면 모델을 활성화한다

### AC-007-02: 모델 무결성 검증

**Given** 모델 다운로드가 완료되었다
**When** 시스템이 모델 파일을 로드한다
**Then** SHA-256 체크섬이 예상값과 일치하는지 검증한다
**And** 체크섬 불일치 시 손상된 파일을 삭제하고 재다운로드를 시도한다

### AC-007-03: 저장 공간 부족 처리

**Given** 기기 남은 저장 공간이 모델 크기의 2배(약 300MB) 미만이다
**When** 모델 다운로드를 시도한다
**Then** 다운로드를 차단하고 "저장 공간이 부족합니다" 안내를 표시한다
**And** 필요 저장 공간 안내와 함께 설정 이동 옵션을 제공한다

### AC-007-04: 이어서 다운로드 (Resumable)

**Given** 모델 다운로드 중 네트워크가 끊겼다 (50% 진행 상태)
**When** 네트워크가 복구되고 앱을 재시작한다
**Then** 50% 지점부터 다운로드를 이어서 진행한다 (처음부터 재시작하지 않음)
**And** 부분 다운로드 파일이 존재하면 이를 활용한다

### AC-007-05: 앱 삭제 시 모델 제거

**Given** 모델이 앱 내부 저장소에 다운로드되어 있다
**When** 사용자가 앱을 삭제한다
**Then** 모델 파일이 앱 내부 저장소에 위치하므로 OS에 의해 자동 제거된다

## 2. REQ-MOBILE-008: 오프라인 STT 처리 엔진

### AC-008-01: 오프라인 STT 기본 처리

**Given** 네트워크가 오프라인 상태이고 모델이 다운로드되어 있다
**When** 사용자가 2분 길이의 녹음을 완료한다
**Then** 시스템이 M4A 오디오를 16kHz mono WAV로 변환한다
**And** whisper.cpp를 통해 로컬 STT 추론을 실행한다
**And** 변환된 텍스트 결과를 화면에 표시한다

### AC-008-02: 긴 오디오 청크 분할 처리

**Given** 네트워크가 오프라인 상태이고 모델이 다운로드되어 있다
**When** 사용자가 10분 길이의 녹음을 완료한다
**Then** 시스템이 오디오를 30초 단위 청크로 분할한다
**And** 각 청크를 순차적으로 처리하여 메모리 사용량을 안정적으로 유지한다
**And** 전체 청크 결과를 병합하여 최종 텍스트를 생성한다

### AC-008-03: 메모리 부족 처리

**Given** 오프라인 STT 처리가 진행 중이다
**When** 기기 메모리 사용량이 임계치(사용 가능 메모리 10% 미만)에 도달한다
**Then** 현재 처리 중인 청크를 완료한 후 처리를 중단한다
**And** 사용자에게 "메모리 부족으로 처리가 중단되었습니다" 안내를 표시한다
**And** 부분 처리 결과가 있으면 표시한다

### AC-008-04: 임시 파일 정리

**Given** 오프라인 STT 처리가 완료되었다
**When** STT 결과가 생성되어 화면에 표시된다
**Then** 전처리로 생성된 임시 WAV 파일을 즉시 삭제한다
**And** 오디오 원본(M4A)은 유지한다 (재처리를 위해)

### AC-008-05: 오프라인 결과 메타데이터

**Given** 오프라인 STT 처리가 완료되었다
**When** 결과 객체를 생성한다
**Then** 결과에 `offline: true` 메타데이터가 포함된다
**And** 오프라인 결과임을 나타내는 배지가 UI에 표시된다

### AC-008-06: STT 처리 진행률

**Given** 오프라인 STT 처리가 진행 중이다
**When** 각 청크 처리가 완료된다
**Then** 전체 진행률이 (완료된 청크 수 / 전체 청크 수)로 계산된다
**And** UI에 0%~100% 진행률이 실시간 업데이트된다

## 3. REQ-MOBILE-009: 하이브리드 온/오프라인 파이프라인

### AC-009-01: 오프라인 시 로컬 STT 실행

**Given** 네트워크가 오프라인 상태이고 모델이 다운로드되어 있다
**When** 사용자가 녹음을 완료한다
**Then** PipelineProvider가 네트워크 상태를 확인하여 오프라인 경로를 선택한다
**And** 로컬 STT 파이프라인이 자동으로 실행된다
**And** 기존 백엔드 업로드는 시도하지 않는다

### AC-009-02: 온라인 시 기존 파이프라인 실행

**Given** 네트워크가 온라인 상태이다
**When** 사용자가 녹음을 완료한다
**Then** PipelineProvider가 기존 백엔드 파이프라인을 실행한다
**And** SSE + 폴링 기반 진행률 업데이트가 동작한다
**And** 오프라인 STT 엔진은 호출되지 않는다

### AC-009-03: 네트워크 복구 시 자동 재처리

**Given** 오프라인 STT 결과(`offline: true`)가 표시되어 있다
**When** 네트워크가 오프라인에서 온라인으로 전환된다
**Then** 시스템이 자동으로 원본 오디오를 백엔드에 업로드한다
**And** whisper-large-v3로 재처리를 요청한다
**And** 재처리 완료 시 오프라인 결과를 온라인 결과로 교체한다

### AC-009-04: 재처리 실패 시 기존 결과 유지

**Given** 네트워크 복구 후 재처리가 진행 중이다
**When** 백엔드 재처리가 실패한다 (서버 오류, 타임아웃 등)
**Then** 기존 오프라인 STT 결과를 유지한다
**And** 사용자에게 "고품질 재처리 실패. 로컬 결과를 유지합니다" 안내를 표시한다
**And** 수동 재시도 버튼을 제공한다

### AC-009-05: 개선된 결과 배지

**Given** 오프라인 결과가 온라인 재처리 결과로 교체되었다
**When** 사용자가 해당 회의록을 조회한다
**Then** "개선된 결과" 배지가 표시된다
**And** 오프라인/온라인 처리 이력을 확인할 수 있다

### AC-009-06: 파이프라인 전환 시 상태 일관성

**Given** 녹음 완료 후 파이프라인이 실행 중이다
**When** 네트워크 상태가 오프라인↔온라인으로 전환된다
**Then** 현재 실행 중인 파이프라인을 안전하게 완료하거나 취소한다
**And** 새로운 네트워크 상태에 맞는 파이프라인을 시작하지 않는다 (이미 실행 중이므로)
**And** 상태 불일치가 발생하지 않는다

## 4. REQ-MOBILE-010: 모델 다운로드 UX

### AC-010-01: Wi-Fi 자동 다운로드

**Given** 모델이 다운로드되지 않았고 기기가 Wi-Fi에 연결되어 있다
**When** 사용자가 오프라인 STT 기능에 접근한다
**Then** 다이얼로그에서 "다운로드" 버튼 탭 시 확인 없이 즉시 다운로드를 시작한다

### AC-010-02: 셀룰러 확인

**Given** 모델이 다운로드되지 않았고 기기가 셀룰러 네트워크에 연결되어 있다
**When** 사용자가 오프라인 STT 기능에 접근한다
**Then** "현재 모바일 데이터를 사용 중입니다. 약 150MB를 다운로드합니다. 계속하시겠습니까?" 확인 다이얼로그를 표시한다
**And** 사용자가 취소하면 다운로드하지 않는다

### AC-010-03: 다운로드 진행률 표시

**Given** 모델 다운로드가 진행 중이다 (50%)
**When** 사용자가 다운로드 화면을 확인한다
**Then** 현재 진행률이 "50%"로 표시된다
**And** 예상 남은 시간이 표시된다 (예: "약 2분 남음")

### AC-010-04: 다운로드 중 네트워크 끊김

**Given** 모델 다운로드가 70% 진행 중이다
**When** 네트워크 연결이 끊긴다
**Then** 다운로드가 일시 정지되고 "네트워크 대기 중" 상태로 표시된다
**And** 네트워크가 복구되면 자동으로 다운로드가 재개된다

### AC-010-05: 다운로드 실패 재시도

**Given** 모델 다운로드가 3회 연속 실패했다
**When** 자동 재시도가 모두 소진된다
**Then** "다운로드 실패" 메시지와 수동 "재시도" 버튼을 표시한다
**And** 사용자가 재시도 버튼을 탭하면 다운로드를 다시 시작한다

## 5. REQ-MOBILE-011: 플랫폼별 STT 통합

### AC-011-01: iOS Core ML 가속

**Given** iOS 기기에서 앱이 실행 중이다
**When** 오프라인 STT 처리를 시작한다
**Then** whisper.cpp가 Core ML 백엔드를 사용하여 Neural Engine(ANE) 가속으로 처리한다
**And** CPU 전용 처리 대비 최소 2배 빠른 처리 속도를 보인다

### AC-011-02: macOS mlx-whisper

**Given** macOS 기기에서 앱이 실행 중이다
**When** 오프라인 STT 처리를 시작한다
**Then** mlx-whisper를 직접 호출하여 MPS 가속으로 처리한다
**And** 기존 백엔드 STT 엔진과 동일한 품질의 결과를 생성한다

### AC-011-03: Android TFLite

**Given** Android 기기에서 앱이 실행 중이다
**When** 오프라인 STT 처리를 시작한다
**Then** whisper.cpp가 TFLite 런타임을 사용하여 ARM NEON 최적화로 처리한다
**And** NNAPI 지원 기기에서는 NNAPI 가속을 활용한다

### AC-011-04: 플랫폼 자동 감지

**Given** 앱이 실행되었다
**When** PlatformSttService가 초기화된다
**Then** 현재 플랫폼(iOS/macOS/Android)을 자동으로 감지한다
**And** 감지된 플랫폼에 적합한 STT 엔진(Core ML/mlx/TFLite)을 로드한다
**And** 지원하지 않는 플랫폼에서는 명확한 에러 메시지를 반환한다

### AC-011-05: Platform Channel 통신

**Given** Flutter PlatformSttService가 초기화되었다
**When** Dart 코드에서 `transcribe(audioPath, language: "ko")`를 호출한다
**Then** MethodChannel을 통해 네이티브 코드로 요청이 전달된다
**And** 네이티브 코드가 처리 완료 후 결과를 Dart로 반환한다
**And** 처리 중 오류 발생 시 PlatformException이 Dart로 전파된다

## 6. 성능 기준 (Performance Criteria)

| 항목 | 기준 | 측정 방법 |
|------|------|----------|
| whisper-base 다운로드 시간 | Wi-Fi 환경에서 60초 이내 (150MB) | 10Mbps 연결 기준 |
| 오프라인 STT 처리 속도 | 실시간의 2배 이상 (5분 녹음 → 2.5분 이내 처리) | iPhone 15 Pro 기준 |
| 16kHz WAV 변환 시간 | 5분 녹음 기준 10초 이내 | 로컬 전처리 |
| 메모리 사용량 | STT 처리 중 500MB 이하 | iOS/Android 프로파일링 |
| 배터리 소모 | 5분 녹음 STT 처리 시 5% 이하 | iPhone 15 Pro 기준 |
| 네트워크 복구 후 재처리 시작 | 온라인 전환 감지 후 5초 이내 | ConnectivityService 이벤트 |

## 7. 품질 게이트 (Quality Gates)

### 기능 검증

- [ ] 오프라인 상태에서 녹음 완료 후 로컬 STT 처리가 정상 동작한다
- [ ] 온라인 상태에서 녹음 완료 후 기존 백엔드 파이프라인이 정상 동작한다
- [ ] 오프라인→온라인 전환 시 자동 재처리가 동작한다
- [ ] 재처리 실패 시 기존 결과 유지 + 수동 재시도 가능하다
- [ ] 모델 다운로드/검증/저장이 정상 동작한다

### 플랫폼 검증

- [ ] iOS에서 Core ML 가속 STT 처리가 동작한다
- [ ] macOS에서 mlx-whisper STT 처리가 동작한다
- [ ] Android에서 TFLite STT 처리가 동작한다
- [ ] 플랫폼 자동 감지가 정확하게 동작한다

### 에지 케이스 검증

- [ ] 모델 미다운로드 상태에서 STT 접근 시 적절한 안내
- [ ] 저장 공간 부족 시 다운로드 차단
- [ ] 다운로드 중 네트워크 끊김 → 재개
- [ ] 10분 초과 긴 오디오 청크 분할 처리
- [ ] 메모리 부족 시 안전한 처리 중단
- [ ] 네트워크 전환 중 파이프라인 상태 일관성 유지

### TRUST 5 검증

- [ ] **Tested**: 신규 서비스별 단위 테스트 85%+ 커버리지
- [ ] **Readable**: 한국어 주석(Why 중심), 명확한 네이밍
- [ ] **Unified**: 기존 Riverpod 패턴 일관성, ruff/flutter analyze 통과
- [ ] **Secured**: 모델 체크섬 검증, 안전한 파일 처리
- [ ] **Trackable**: Conventional commits, SPEC-MOBILE-002 참조
