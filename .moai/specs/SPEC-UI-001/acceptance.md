# SPEC-UI-001: 인수 기준

## AC-001: 양식 선택 시 동적 테이블 표시

**Given** PDF 양식(예: 과정명/프로젝트명/팀명/참석자/회의안건/회의내용/결정된 사안)을 업로드하고 선택한 상태에서
**When** 녹음 후 전체 파이프라인이 완료되어 회의록 탭을 열면
**Then** 양식의 섹션 구조에 맞는 테이블이 동적으로 생성되어 각 섹션에 AI가 정리한 내용이 표시된다

## AC-002: 다른 양식 선택 시 다른 테이블 구조

**Given** 다른 형태의 양식(예: 회의 목적/논의 사항/합의 내용/후속 조치)을 업로드하고 선택한 상태에서
**When** 녹음 후 전체 파이프라인이 완료되면
**Then** 해당 양식의 섹션 구조에 맞는 다른 테이블이 표시된다

## AC-003: 양식 미선택 시 기본 테이블 유지

**Given** 양식을 선택하지 않은 상태에서
**When** 녹음 후 전체 파이프라인이 완료되면
**Then** 기존 하드코딩된 기본 테이블(과정명/프로젝트명/팀명/작성자/참석자/회의안건/회의내용/결정된 사안)이 표시된다

## AC-004: 기존 결과 하위 호환성

**Given** SPEC-UI-001 이전에 생성된 기존 회의 결과가 있을 때
**When** 해당 결과의 회의록 탭을 열면
**Then** 기본 테이블이 정상 표시된다 (template_structure 없는 경우 fallback)

## 품질 기준

- 기존 테스트 전체 통과 (회귀 없음)
- 새 양식 업로드 → 녹음 → 결과 확인 E2E 테스트 통과
