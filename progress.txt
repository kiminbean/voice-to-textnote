# Voice-to-TextNote Backend Optimization Progress

## Completed Changes

### BUG-017 ✅ In-flight worker status drops ownership metadata
- **Files:** `backend/workers/tasks/status_context.py`, all worker task status updaters
- **Fix:** Added `merge_existing_status_context()` and wired STT/DIA/minutes/summary/sentiment/tone/mind-map status updates to preserve `user_id`, guest session metadata, parent task ids, and `created_at`. This prevents authenticated in-flight status polling from returning false 404s before DB persistence finishes.
- **Verification:** `backend/tests/unit/test_worker_status_context.py`, `backend/tests/unit/test_tone_task.py`, `backend/tests/unit/test_diarization_task_v2.py` → `43 passed`; broader worker set → `138 passed`; ruff clean.

### BUG-018 ✅ Derived task access did not fall back to parent task ownership
- **Files:** `backend/app/dependencies.py`, `backend/tests/unit/test_dependencies.py`
- **Fix:** `has_task_access()` now checks parent ids in payload (`source_task_id`, `stt_task_id`, `diarization_task_id`, `minutes_task_id`, `summary_task_id`) before denying access. This fixes derived mind-map/status lookups where the derived task has no separate ownership row yet.
- **Verification:** `backend/tests/unit/test_dependencies.py backend/tests/unit/test_worker_status_context.py` → `34 passed`; ruff clean.

### BUG-019 ✅ Tone analysis failure polluted DIA task stream
- **File:** `backend/workers/tasks/tone_task.py`
- **Fix:** Tone status updates no longer publish generic task SSE events by default. Optional tone failures remain isolated from the DIA/minutes pipeline.
- **Verification:** `backend/tests/unit/test_tone_task.py` included in worker regression runs.

### OPS-001 ✅ Local Tailscale server deployment clarified
- **Server:** `100.69.69.119:8000` is the current Mac's Tailscale address, not a separate remote host.
- **Runtime:** API and Celery worker run in tmux session `voice-to-textnote-server`.
- **Verification:** `/api/v1/health` reports API, Redis, Celery worker, and ffmpeg healthy.

### SPEAKER-001 ✅ Saved speaker names are reused in later minutes generation
- **Files:** `backend/workers/tasks/minutes_task.py`, `backend/pipeline/minutes_formatter.py`, `backend/db/speaker_models.py`
- **Fix:** Minutes generation now loads the authenticated user's global `SpeakerProfile` names and merges them with request-level `speaker_names`. Saved speakers still count toward automatic numbering, so if `SPEAKER_00` is named `영자`, the next unnamed speaker becomes `Speaker 2`, not another `Speaker 1`.
- **Verification:** `backend/tests/unit/test_minutes_task.py backend/tests/unit/test_minutes_formatter.py backend/tests/unit/test_speakers_api.py` → `51 passed`; ruff clean.
- **Follow-up completed:** True voiceprint matching across arbitrary diarization labels was implemented on 2026-06-24 under "화자 목소리 기억/자동 이름 적용".

### BUG-001 ✅ transcription_task _update_task_status pipeline race
- **File:** `backend/workers/tasks/transcription_task.py`
- **Fix:** Replaced the buggy GET+SETEX pipeline (which wrote without created_at first, then conditionally overwrote) with a simple GET → build dict → single SETEX. Now always preserves or omits created_at correctly in one write.

### BUG-002 ✅ upload_transcription active_job_count race condition
- **File:** `backend/app/api/v1/transcription.py`
- **Fix:** Replaced the INCR+DECR pipeline (which created phantom increments) with a simple ZCARD query on the same `active_jobs_ts` sorted set that the worker uses. No mutation on read.

### BUG-003 ✅ file_size_bytes undefined variable
- **File:** `backend/app/api/v1/transcription.py`
- **Fix:** Changed `file_size_bytes` to `file_size` in the initial_status dict (NameError at runtime).

### BUG-004 ✅ lifecycle.py creates separate DB engine
- **File:** `backend/app/lifecycle.py`
- **Fix:** Both validate_startup() and cleanup_shutdown() now import `_db_engine` from `backend.app.dependencies` instead of calling `create_engine()`. This ensures table creation happens on the actual app engine and dispose() cleans up the real pool.

### BUG-005 ✅ Redis client leak in validate_startup
- **File:** `backend/app/lifecycle.py`
- **Fix:** validate_startup now uses the shared `get_redis_client()` from dependencies.py instead of creating a new connection with `aioredis.from_url()`.

### BUG-006 ✅ Duplicated Redis client singleton across workers
- **File:** New `backend/workers/redis_client.py`, updated all task files
- **Fix:** Created a shared `get_worker_redis()` function with a ConnectionPool. All 5 worker task files (transcription, summary, diarization, minutes, sentiment) now use this shared pool instead of independent `redis.from_url()` calls.

### BUG-008 ✅ Active job tracking not crash-resilient
- **Files:** `backend/workers/tasks/transcription_task.py`, `summary_task.py`, `diarization_task.py`
- **Fix:** Replaced plain SET+INCR tracking with Redis sorted sets scored by timestamp. On each read, entries older than 2 hours (exceeding the 65-min hard timeout) are pruned. This prevents orphan entries from permanently reducing capacity if a worker crashes.

### PERF-001 ✅ tagging_engine creates new httpx client per call
- **File:** `backend/ml/tagging_engine.py`
- **Fix:** Added module-level `_http_client` singleton with lazy init. `generate_auto_tags` reuses the client across calls instead of creating a new `AsyncClient` per invocation.

### PERF-002 ✅ transcription API reads entire file into memory
- **File:** `backend/app/api/v1/transcription.py`
- **Fix:** Changed upload to stream chunks (1MB) to disk, then validate size from file stats. Eliminates loading up to 500MB into Python memory.

### SEC-001 ✅ temp_path.write_bytes has no error handling
- **File:** `backend/app/api/v1/transcription.py`
- **Fix:** Replaced write_bytes with streaming write in try/except IOError. On failure, temp file is cleaned up before returning 422.

### BUG-010 ✅ AuthService bcrypt/passlib runtime failure with bcrypt 5
- **File:** `backend/db/auth_service.py`
- **Fix:** Replaced passlib bcrypt hashing with direct bcrypt + SHA-256 prehashing for new passwords, while preserving verification for legacy raw bcrypt hashes. This avoids bcrypt 5's 72-byte password ValueError and passlib backend detection failure.

### BUG-011 ✅ tagging_engine shared httpx client NameError and shutdown leak
- **Files:** `backend/ml/tagging_engine.py`, `backend/app/lifecycle.py`
- **Fix:** Added the missing module-level httpx import, removed the function-local import mismatch, and added shutdown cleanup for the shared AsyncClient.

### BUG-012 ✅ Webhook URL validation was ineffective
- **Files:** `backend/schemas/webhook.py`, `backend/utils/validators.py`, `backend/db/webhook_service.py`, `backend/services/webhook_notifier.py`
- **Fix:** Replaced the ineffective `Annotated[str, AnyHttpUrl]` usage with explicit URL normalization/validation. Invalid schemes, malformed URLs, localhost, private IPs, link-local metadata IPs, and private DNS resolutions are rejected before webhook sends.

### BUG-013 ✅ audio_analysis RMS dBFS calculation and sample-rate scoring
- **File:** `backend/ml/audio_analysis_engine.py`
- **Fix:** Corrected RMS dBFS to use `20 * log10(rms / max_amplitude)` and reordered sample-rate checks so sub-8kHz audio receives the intended stronger quality penalty.

### BUG-014 ✅ audio_analysis upload reads full file before enforcing size
- **File:** `backend/app/api/v1/audio_analysis.py`
- **Fix:** Changed upload handling to stream chunks to a temp file while enforcing the size limit, preventing large uploads from being fully loaded into memory.

### BUG-015 ✅ App shutdown leaves shared Redis client open
- **Files:** `backend/app/dependencies.py`, `backend/app/lifecycle.py`
- **Fix:** Added `close_redis_client()` and wired it into shutdown cleanup without creating a Redis client if one was never initialized.

### BUG-016 ✅ Alembic local package shadows external alembic library
- **Files:** `alembic/__init__.py`, `alembic/versions/__init__.py`
- **Fix:** Removed empty package marker files so `from alembic import context/op` resolves to the installed Alembic library. Verified `alembic heads` returns `001 (head)`.

## Not Fixed (documented/triaged)

### BUG-007 chunk_manager split_audio loads entire audio into memory
- **Status:** Documented limitation. For 4-hour audio at 16kHz mono, ~460MB is loaded. pydub AudioSegment requires full in-memory load. Streaming chunking would require switching to pydub-less chunking (e.g., ffmpeg subprocess). Marked as future optimization.

### BUG-009 Redundant SoftTimeLimitExceeded handler
- **Status:** Defensive code — the inner handler returns a dict, the outer wrapper's handler would only trigger if the inner function somehow re-raised. Left as-is (not harmful).

### QUAL-001 Class-level Lock for singletons
- **Status:** Not a bug — Python class-level mutable attributes are shared across all references. The double-checked locking pattern works correctly.

## Tests Updated
- `backend/tests/unit/test_lifecycle.py` — Updated all mocks to patch `backend.app.dependencies` instead of the removed `backend.app.lifecycle.create_engine` and `redis.asyncio.from_url`.
